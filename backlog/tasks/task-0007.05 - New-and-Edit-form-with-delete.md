## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
EditController GET /new (empty form), GET /edit/{slug}/{version} (pre-filled), POST /save (create/update), POST /delete/{slug}/{version}. new.html and edit.html with Monaco editor for content. Requires ROLE_EDITOR.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 New form saves new artifact and redirects to detail page
- [ ] #2 Edit form pre-fills existing content and metadata
- [ ] #3 Delete button removes artifact and redirects to browse
- [ ] #4 Validation errors shown inline
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create src/main/java/com/agentlibrary/web/EditController.java in package com.agentlibrary.web:
   - @Controller, @PreAuthorize("hasRole(\"EDITOR\")") at class level
   - Constructor-injected with ArtifactService
   - @GetMapping("/new") → returns "edit" view with empty form model
   - @GetMapping("/edit/{slug}/{version}") → returns "edit" view with pre-filled form model
   - @PostMapping("/save") → processes form submission, redirects to artifact detail
   - @PostMapping("/delete/{slug}/{version}") → deletes artifact, redirects to /browse
2. Create src/main/java/com/agentlibrary/web/ArtifactForm.java — form-backing object:
   - Fields: name (String), title (String), type (String), version (String), description (String), harnesses (List<String>), tags (String — comma-separated), category (String), language (String), author (String), visibility (String), content (String — the body content edited in Monaco)
   - Jakarta Validation annotations: @NotBlank on name, type, version; @Pattern on name for slug format
   - Method toRawContent(): reconstructs the full artifact string with YAML frontmatter + body content (using MetadataCodec.encode() or manual YAML building)
3. Implement GET /new handler:
   - Create empty ArtifactForm with sensible defaults (version="1.0.0", visibility="team")
   - Add "form" to model, "isNew" = true, "pageTitle" = "New Artifact"
   - Add ArtifactType.values() as "types" and Harness.values() as "harnesses" for dropdowns
   - Return "edit"
4. Implement GET /edit/{slug}/{version} handler:
   - Fetch artifact: artifactService.get(slug, version)
   - Populate ArtifactForm from artifact metadata and content
   - Tags: join list with ", " for display in text field
   - Harnesses: set as selected checkboxes
   - Add "form" to model, "isNew" = false, "pageTitle" = "Edit: " + title
   - Return "edit"
5. Implement POST /save handler:
   - Accept @Valid @ModelAttribute ArtifactForm, BindingResult, RedirectAttributes, Principal
   - If bindingResult.hasErrors(): return "edit" with form and errors for inline display
   - Convert form to raw content string (frontmatter + body)
   - Extract User from Principal (reuse pattern from ArtifactApiController.extractUser)
   - Call artifactService.createOrUpdate(rawContent, user)
   - Catch ValidationException: add field errors to model, return "edit"
   - On success: add flash attribute "success" = "Artifact saved successfully"
   - Redirect to /artifacts/{slug}/{version}
6. Implement POST /delete/{slug}/{version} handler:
   - @PreAuthorize("hasRole(\"EDITOR\")") (inherited from class)
   - Extract User from Principal
   - Call artifactService.delete(slug, user)
   - Add flash "success" = "Artifact deleted"
   - Redirect to /browse
7. Create src/main/resources/templates/edit.html using layout.html decorator:
   - Form with th:object="${form}" th:action="@{/save}" method="post"
   - Metadata fields: name (text input, readonly if !isNew), title, type (select from types), version (text input)
   - Description textarea
   - Harnesses: checkboxes for each Harness value
   - Tags: text input (comma-separated)
   - Category, language, author, visibility (select: team/private) inputs
   - Content area: hidden textarea (th:field="*{content}") synced with Monaco editor
   - Monaco editor div: initialised in page-specific scripts block using Monaco CDN loader
   - Validation errors: th:errors="*{fieldName}" displayed as inline red text below each field
   - Buttons row: "Save" submit button, "Delete" button (only if !isNew, styled as danger, confirms via JS dialog)
   - Delete form: separate <form th:action="@{/delete/{slug}/{version}(slug=${form.name}, version=${form.version})}" method="post"> with confirmation
8. Monaco editor integration (in layout:fragment="scripts"):
   - Load Monaco from CDN: require.config paths to CDN, require monaco editor
   - Initialise editor on div#monaco-editor with language "markdown" (or detect from type)
   - Sync editor content to hidden textarea on form submit (addEventListener on form submit event)
   - Set editor value from hidden textarea content on page load (for edit mode)
9. Helper method in EditController — extractUser(Principal):
   - Same pattern as API controller: check Authentication, extract roles, return User record
10. Test approach — create EditControllerTest.java:
    - @WebMvcTest(EditController.class) with @MockBean ArtifactService, Bucket
    - Test GET /new with EDITOR role returns 200, view "edit", model has empty form and isNew=true
    - Test GET /new without EDITOR role returns 403
    - Test GET /edit/{slug}/{version} with EDITOR role returns 200, form pre-filled
    - Test POST /save with valid data: verify artifactService.createOrUpdate called, redirect to detail
    - Test POST /save with invalid data (blank name): returns 200 (form page) with errors
    - Test POST /save with ValidationException: returns form with error message
    - Test POST /delete/{slug}/{version}: verify artifactService.delete called, redirect to /browse
    - Test all endpoints reject USER role (403)
11. AC mapping:
    - AC#1 (new form saves and redirects): POST /save → createOrUpdate → redirect /artifacts/{slug}/{version}
    - AC#2 (edit form pre-fills): GET /edit populates ArtifactForm from existing artifact
    - AC#3 (delete button removes and redirects): POST /delete → delete → redirect /browse
    - AC#4 (validation errors inline): BindingResult + th:errors inline display
12. Dependencies: Requires TASK-0007.01 (layout.html, Monaco setup). Independent of TASK-0007.02/03/04.
