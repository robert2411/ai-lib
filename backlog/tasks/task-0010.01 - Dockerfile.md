## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create `.dockerignore` at project root excluding: target/, .git/, .idea/, *.iml, node_modules/, backlog/, docs/, .env, docker-compose.yml
2. Create `Dockerfile` with two stages:
   a. BUILD stage — FROM eclipse-temurin:21-jdk AS build
      - WORKDIR /build
      - COPY pom.xml mvnw mvnw.cmd ./
      - COPY .mvn .mvn
      - RUN chmod +x mvnw && ./mvnw dependency:go-offline -B
      - COPY src src
      - RUN ./mvnw package -DskipTests -B
   b. RUNTIME stage — FROM eclipse-temurin:21-jre-alpine
      - RUN apk add --no-cache git bash curl jq
      - RUN addgroup -S appgroup && adduser -S appuser -G appgroup
      - WORKDIR /app
      - COPY --from=build /build/target/agent-library-*.jar app.jar
      - COPY scripts/gen-password-hash.sh /app/scripts/
      - RUN chmod +x /app/scripts/gen-password-hash.sh
      - RUN mkdir -p /data && chown -R appuser:appgroup /data
      - ENV LIBRARY_DATA_DIR=/data
      - ENV LIBRARY_REPO_PATH=/data/library.git
      - ENV LIBRARY_WORK_PATH=/data/library-work
      - ENV LIBRARY_USERS_FILE=/data/users.yaml
      - VOLUME /data
      - EXPOSE 8080
      - USER appuser
      - HEALTHCHECK --interval=30s --timeout=5s --retries=3 CMD curl -f http://localhost:8080/actuator/health || exit 1
      - ENTRYPOINT ["java", "-jar", "app.jar"]
3. Add `spring-boot-starter-actuator` dependency to pom.xml (no version — managed by parent BOM)
4. Configure actuator in application.yml — expose only health endpoint:
   management.endpoints.web.exposure.include=health
   management.endpoint.health.show-details=never
5. Update SecurityConfig.java — add `/actuator/health` to permitAll() alongside existing `/healthz`
6. Verify: run `docker build -t agent-library:latest .` completes without errors (AC #1)
7. Verify: run container with `-v test-vol:/data -p 8080:8080`, hit http://localhost:8080/actuator/health returns {"status":"UP"} (AC #2)
8. Verify: stop container, restart with same volume, confirm /data/library.git persists (AC #3)

🔍 PLAN REVIEW CONCERNS:
- Concern #1: ENTRYPOINT hardcodes --library.data-dir=/data, --library.repo-path, --library.work-path, --library.users-file as command-line args. Spring Boot precedence: CLI args > env vars > application.yml. This renders docker-compose env vars (TASK-0010.02) completely inert — they can never override. Design doc (§Docker Deployment) shows env-var-driven config in docker-compose, not hardcoded ENTRYPOINT args.
  Fix: Remove --library.* args from ENTRYPOINT. Use ENV directives in Dockerfile to set defaults (ENV LIBRARY_DATA_DIR=/data, etc.). ENTRYPOINT becomes ["java", "-jar", "app.jar"]. This keeps /data as the default for standalone runs, while allowing docker-compose env vars to override if needed.

Verdict: Plan needs revision — ENTRYPOINT/env-var precedence conflict must be resolved before implementation.
