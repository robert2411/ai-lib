## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create src/main/java/com/agentlibrary/web/BrowseController.java in package com.agentlibrary.web:
   - @Controller, constructor-injected with ArtifactService, IndexService, FacetService
   - @GetMapping("/browse") → full page render, returns "browse" view name
   - @GetMapping("/browse/results") → fragment-only response for HTMX swap, returns "fragments/browse-results :: results"
   - Both methods accept @RequestParam: q (String, optional), type (String, optional), harness (String, optional), tag (String, optional), page (int, default 0), size (int, default 20)
2. Implement shared logic (private method buildBrowseModel):
   - Parse type → ArtifactType.fromSlug(type) if non-null
   - Parse harness → Harness.fromSlug(harness) if non-null
   - Build Filter(artifactType, harnessEnum, tag, q)
   - If q is non-null: use artifactService.search(q, filter) with pagination
   - Else: use artifactService.list(filter, page, size) and artifactService.count(filter)
   - Add to model: "artifacts" (List<ArtifactMetadata>), "currentPage", "totalPages", "totalCount"
   - Add filter state to model: "selectedType", "selectedHarness", "selectedTag", "query"
3. Full page GET /browse handler:
   - Call buildBrowseModel
   - Add facet lists to model: facetService.getDistinctTypes(), getDistinctHarnesses(), getDistinctTags()
   - Return "browse"
4. Fragment GET /browse/results handler:
   - Detect HTMX request via "HX-Request" header (check request.getHeader("HX-Request") != null)
   - Call buildBrowseModel
   - Return "fragments/browse-results :: results" (Thymeleaf fragment expression)
   - If NOT an HTMX request (direct navigation), redirect to /browse with same params
5. Create src/main/resources/templates/browse.html using layout.html decorator:
   - Left sidebar (or top bar) with filter controls:
     * Type checkboxes: iterate over types list, each checkbox has hx-get="/browse/results" hx-target="#results" hx-include="[name=type],[name=harness],[name=tag],[name=q]" hx-trigger="change"
     * Harness checkboxes: same pattern
     * Tag checkboxes (or multi-select): same pattern
     * Ensure agent-group appears as a selectable type (AC#5)
   - Search input (pre-filled with q value) with hx-get trigger on keyup with delay:300ms
   - Results area with id="results" containing th:replace of browse-results fragment
   - Pagination at bottom
6. Create src/main/resources/templates/fragments/browse-results.html:
   - th:fragment="results" wrapping:
     * Artifact card grid using th:each and fragments/artifact-card.html
     * For agent-group cards: show a role count badge (e.g. "3 members" or role breakdown) — AC#6
     * Empty state: th:if="${#lists.isEmpty(artifacts)}" show "No artifacts match your filters" message
     * Pagination fragment inclusion with filter params preserved in URLs
7. Pagination implementation:
   - Calculate totalPages from totalCount / size (ceiling division)
   - Pagination links include all current filter params as query string: ?type=X&harness=Y&tag=Z&q=W&page=N
   - For HTMX: pagination links use hx-get with hx-target="#results" to swap without full page reload
8. Test approach — create BrowseControllerTest.java:
   - @WebMvcTest(BrowseController.class) with mocks for ArtifactService, IndexService, FacetService, Bucket
   - Test GET /browse returns 200, view "browse", model contains artifacts and facets
   - Test GET /browse?type=skill filters correctly (verify artifactService called with correct filter)
   - Test GET /browse/results with HX-Request header returns fragment (status 200, no full layout)
   - Test GET /browse/results without HX-Request header redirects to /browse
   - Test pagination: page=1 returns second page of results
   - Test empty results: returns 200 with empty artifacts list
   - Test agent-group type appears in types list
9. AC mapping:
   - AC#1 (full page with filters and results): browse.html with sidebar + results area
   - AC#2 (checkbox triggers HTMX swap): hx-get + hx-target on filter controls
   - AC#3 (pagination with filter state): query params preserved in pagination links
   - AC#4 (no results empty state): th:if check in browse-results fragment
   - AC#5 (agent-group as selectable type): included in types from FacetService (which reads from index)
   - AC#6 (agent-group cards show member role count badge): conditional badge in artifact-card
10. Dependencies: Requires TASK-0007.01 (layout.html, fragments). No dependency on TASK-0007.02.
