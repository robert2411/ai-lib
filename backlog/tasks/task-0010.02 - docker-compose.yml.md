## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create `docker-compose.yml` at project root with:
   services:
     agent-library:
       build:
         context: .
         dockerfile: Dockerfile
       image: agent-library:latest
       ports:
         - "8080:8080"
       volumes:
         - library-data:/data
       environment:
         SPRING_PROFILES_ACTIVE: prod
         LIBRARY_DATA_DIR: /data
         LIBRARY_REPO_PATH: /data/library.git
         LIBRARY_WORK_PATH: /data/library-work
         LIBRARY_USERS_FILE: /data/users.yaml
       restart: unless-stopped
       healthcheck:
         test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
         interval: 30s
         timeout: 5s
         retries: 3
   volumes:
     library-data:
       name: agent-library-data
2. Map environment variables to Spring Boot properties using relaxed binding:
   - LIBRARY_DATA_DIR -> library.data-dir
   - LIBRARY_REPO_PATH -> library.repo-path
   - LIBRARY_WORK_PATH -> library.work-path
   - LIBRARY_USERS_FILE -> library.users-file
   Note: Spring Boot auto-maps LIBRARY_DATA_DIR to library.data-dir via relaxed binding
3. Verify: `docker compose up --build` builds image and starts service (AC #1)
4. Verify data persistence (AC #2):
   a. `docker compose up -d`
   b. Wait for healthy, push an artifact via API
   c. `docker compose down` (without -v)
   d. `docker compose up -d`
   e. Confirm artifact still present (named volume preserves /data)
