#### 1. `src/main/java/com/agentlibrary/storage/ArtifactRepository.java`

Interface defining the storage contract. Methods match the design doc exactly:

```java
package com.agentlibrary.storage;

import com.agentlibrary.model.*;
import java.io.InputStream;
import java.util.List;
import java.util.Optional;

public interface ArtifactRepository {
    List<ArtifactMetadata> list(Filter filter);
    Optional<Artifact> get(String type, String name, String ref);
    CommitResult save(Artifact artifact, String message, User user);
    void delete(String type, String name, String message, User user);
    List<VersionRef> versions(String type, String name);
    InputStream bundle(String type, String name, String ref);
}
```

Notes:
- `list` returns `List<ArtifactMetadata>` (not full Artifact — callers filter on metadata)
- `get` takes `ref` as nullable String: null/empty means "latest", otherwise a semver or "HEAD"
- `bundle` takes type, name, and ref — serves a single artifact at a specific version as a tar.gz stream. Matches the REST endpoint `GET /artifacts/{type}/{name}/bundle?ref=…`
- `save` returns `CommitResult` with commitId, timestamp, message
- `delete` is void — throws if artifact not found

#### 2. `src/main/java/com/agentlibrary/storage/WriteQueue.java`

Single-writer queue serialising all write operations. Uses a single-threaded executor internally.

```java
package com.agentlibrary.storage;

import java.util.concurrent.*;
import java.util.function.Supplier;

public class WriteQueue {
    private final ExecutorService executor;
    private final long timeoutSeconds;

    public WriteQueue() { this(30); }
    public WriteQueue(long timeoutSeconds) {
        this.timeoutSeconds = timeoutSeconds;
        this.executor = Executors.newSingleThreadExecutor(
            r -> { Thread t = new Thread(r, "write-queue"); t.setDaemon(true); return t; }
        );
    }

    public <T> T submit(Supplier<T> task) { ... }
    public void submit(Runnable task) { ... }
    public void shutdown() { ... }
}
```

Implementation details:
- `submit(Supplier<T>)` wraps in `CompletableFuture`, submits to executor, calls `.get(timeoutSeconds, SECONDS)`
- Propagates checked exceptions wrapped in a `StorageException` (unchecked)
- `submit(Runnable)` delegates to Supplier variant returning null
- `shutdown()` calls `executor.shutdown()` — used in `@PreDestroy` lifecycle
- Thread named "write-queue" for diagnostics
- Timeout prevents deadlocks: throws `StorageException` if task exceeds limit

### Test Approach

#### `src/test/java/com/agentlibrary/storage/WriteQueueTest.java`

Test cases:
1. **Single task executes and returns result** — submit Supplier, verify return value
2. **Tasks execute sequentially** — submit N tasks that record order, verify FIFO order
3. **Concurrent submitters are serialised** — use CountDownLatch, submit from multiple threads, assert no interleaving
4. **Exception in task propagates as StorageException** — submit Supplier that throws, assert StorageException is thrown to caller
5. **Timeout produces StorageException** — create WriteQueue(1) with 1-second timeout, submit task that sleeps 5s, assert timeout exception
6. **shutdown() prevents new submissions** — shutdown then submit, expect RejectedExecutionException wrapped in StorageException

No mocking. Pure unit tests with real concurrency primitives.
