## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
IndexService loads all ArtifactMetadata from INDEX.yaml into an in-memory list on startup. Exposes refresh() called by repository after each commit. Thread-safe read/write via ReadWriteLock.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 IndexService.getAll() returns all artifacts after load
- [ ] #2 refresh() re-reads INDEX.yaml and updates in-memory list
- [ ] #3 Concurrent reads during refresh do not throw
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create package `com.agentlibrary.index` under `src/main/java/com/agentlibrary/index/`.

2. Create `IndexService.java` as a Spring `@Service` bean:
   - Field: `private volatile List<ArtifactMetadata> cache = List.of();`
   - Field: `private final ReentrantReadWriteLock lock = new ReentrantReadWriteLock();`
   - Field: `private final ArtifactRepository repository;` (injected via constructor)
   - Constructor takes `ArtifactRepository` (Spring DI).

3. Implement `@PostConstruct public void init()`:
   - Calls `loadFromIndex()` to populate the cache on startup.

4. Implement `private List<ArtifactMetadata> loadFromIndex()`:
   - Calls `repository.list(null)` to get all artifacts from INDEX.yaml (delegates to the existing LocalGitArtifactRepository which reads INDEX.yaml via `readFileFromHead`).
   - Returns an unmodifiable list.
   - NOTE: The repository.list(null) already reads INDEX.yaml and parses it via IndexFile.load(). The IndexService caches this result so consumers don't re-read git on every call.\n\n5. Implement `public List<ArtifactMetadata> getAll()`:\n   - Acquires `lock.readLock()` in try-finally.\n   - Returns current `cache` reference (already unmodifiable).\n   - AC #1: getAll() returns all artifacts after load.\n\n6. Implement `public void refresh()`:\n   - Acquires `lock.writeLock()` in try-finally.\n   - Calls `loadFromIndex()` and assigns result to `cache`.\n   - Called externally after each commit (from repository or wiring layer).\n   - AC #2: refresh() re-reads INDEX.yaml and updates in-memory list.\n\n7. Thread-safety design (AC #3):\n   - ReadWriteLock ensures multiple concurrent readers can proceed.\n   - Writers (refresh) block readers briefly only during the reference swap.\n   - Actually, since we load data BEFORE acquiring write lock we can minimise lock hold time:\n     a. Load new list outside the lock.\n     b. Acquire writeLock.\n     c. Swap `cache` reference.\n     d. Release writeLock.\n   - This means concurrent reads during refresh do not throw — they see either old or new data.\n\n8. Create `IndexConfig.java` (or add to existing config):\n   - `@Configuration` class in `com.agentlibrary.index`.\n   - Defines `@Bean IndexService` if not using `@Service` annotation directly.\n   - Decision: Use `@Service` on IndexService + `@ComponentScan` picks it up. No separate config needed.\n\n9. Create test `src/test/java/com/agentlibrary/index/IndexServiceTest.java`:\n   - Unit test with mocked `ArtifactRepository`.\n   - Test: `getAll()` returns all artifacts after init (AC #1).\n   - Test: `refresh()` picks up new artifacts added to repo (AC #2).\n   - Test: concurrent reads during refresh don't throw (AC #3) — use CountDownLatch + ExecutorService with multiple reader threads calling getAll() while another thread calls refresh().\n\n10. Files to create:\n    - `src/main/java/com/agentlibrary/index/IndexService.java`\n    - `src/test/java/com/agentlibrary/index/IndexServiceTest.java`\n\n11. Integration note: The existing `LocalGitArtifactRepository.list(null)` reads INDEX.yaml on every call. IndexService caches this, meaning downstream consumers (SearchService, controllers) should call IndexService.getAll() instead of repository.list() for read operations. The repository.save()/delete() methods will trigger IndexService.refresh() (wired in TASK-0004.03).
