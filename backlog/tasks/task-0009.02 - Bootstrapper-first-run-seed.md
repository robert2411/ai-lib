## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Bootstrapper @Component runs on ApplicationReadyEvent. If repo is empty, seeds the agent-library-install skill from skills-seed/ directory into the live repo. Includes skill.md, agent-lib.sh, agent-lib-common.sh, README.md.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Empty repo on first startup contains the seed skill after boot
- [ ] #2 Subsequent startups do not re-seed or overwrite existing content
- [ ] #3 Seed artifacts are queryable via search immediately after boot
- [ ] #4 First-run seed includes an example agent-group artifact (e.g. dev-team-group) with manager, analyse, and implementation member roles

7. Create Bootstrapper component: src/main/java/com/agentlibrary/bootstrap/Bootstrapper.java
   - @Component, implements ApplicationListener<ApplicationReadyEvent>
   - Inject: ArtifactService, IndexService, AppProperties
   - onApplicationEvent():
     a. Check if repo is empty: indexService.getAll().isEmpty()
     b. If empty -> call seedAll()
     c. If not empty -> log "Repo already seeded, skipping bootstrap" and return
   - Maps to AC #2 (idempotency — subsequent startups skip)

8. Implement seedAll() method in Bootstrapper with per-seed error isolation
   - Define list of seed resource paths (ORDER MATTERS — individuals first, group last):
     List.of("seeds/agent-library-install.md", "seeds/agent-lib-sh.md",
             "seeds/agent-lib-common-sh.md", "seeds/agent-lib-readme.md",
             "seeds/dev-team-group.md")
   - For each resource path:
     a. Read classpath resource via getClass().getResourceAsStream("/" + path)
     b. Read full content as UTF-8 string
     c. try {
          artifactService.createOrUpdate(content, SYSTEM_USER);
          log.info("Seeded artifact: {}", path);
        } catch (Exception e) {
          log.warn("Failed to seed {}: {} — continuing with remaining seeds", path, e.getMessage());
          // Do NOT rethrow — continue processing remaining seed files
        }
   - SYSTEM_USER: new User("system", Set.of("ADMIN"))
   - Order: seed individual artifacts BEFORE agent-group so that group member validation
     passes (group references agent-lib-sh, agent-lib-common-sh, agent-lib-readme which
     are already committed by the time dev-team-group is processed)
   - Per-seed error isolation ensures one bad seed file does not abort entire bootstrap

9. Handle group validation — use standard ArtifactService.createOrUpdate() path
   - DECISION: Use ArtifactService.createOrUpdate() for ALL seeds including the group
   - The MetadataValidator must accept type=agent-group. Check MetadataValidator code:
     * If it already handles agent-group type -> no changes needed
     * If it rejects agent-group -> add agent-group to the allowed types list
   - Group member validation (validateGroupMembers) will PASS because member slugs
     (agent-lib-sh, agent-lib-common-sh, agent-lib-readme) are already seeded in
     the same seedAll() run before the group file is processed
   - No new methods, no unchecked bypass, no direct repository.save() calls
   - Keeps Bootstrapper using the standard service API path

10. Ensure search indexes are rebuilt after seeding
    - After all seeds committed, trigger indexService.refresh() explicitly
    - Or rely on repository commit callback (already wired via IndexConfig)
    - Verify: each save() via ArtifactService triggers refreshCallback -> IndexService.refresh() -> SearchService.rebuild()
    - The existing wiring should handle this: save -> commit -> callback -> refresh
    - After seeding, artifacts should be immediately searchable
    - Maps to AC #3

11. Spring bean ordering considerations
    - Bootstrapper uses ApplicationReadyEvent (fires after ALL beans initialised and CommandLineRunners complete)
    - This means IndexConfig CommandLineRunner (which wires refresh callback) runs BEFORE Bootstrapper
    - So by the time Bootstrapper seeds, the callback is already wired -> index will auto-refresh
    - Confirm ordering: @PostConstruct (repo init, index init) -> CommandLineRunner (wire callback) -> ApplicationReadyEvent (bootstrap seed)

12. Testing strategy

      a. Test that onApplicationEvent seeds when index is empty
      b. Test that onApplicationEvent skips when index is non-empty
      c. Test that one failed seed does not prevent others from being seeded (error isolation)
      d. Mock ArtifactService and IndexService
    - Integration test: BootstrapperIntegrationTest (@SpringBootTest with temp git dir)
      a. Start app with empty repo -> verify seed artifacts exist and are searchable
      b. Restart app (same repo) -> verify no duplicate commits / no overwrite
    - Verify all seed files parse correctly via MetadataCodec.decode() in a unit test
    - Test that search("agent-library-install") returns results after boot
    - Verify dev-team-group is seeded successfully with all 3 members
    - Maps to AC #1, #2, #3, #4
