## Implementation Notes

Self-review complete. AC coverage verified:
- AC #1 (empty repo seeded after boot): Steps 7-8 — Bootstrapper checks isEmpty, seeds all artifacts
- AC #2 (no re-seed on subsequent starts): Step 7 — isEmpty check returns false → early return
- AC #3 (searchable immediately): Step 10 — commit callback wires auto-refresh; ApplicationReadyEvent ordering ensures callback is wired before seed runs
- AC #4 (example agent-group with roles): Step 6 — dev-team-group.md with manager/analyse/implementation members

Key design decision: Group member validation bypass during seed.
Preferred approach: Add ArtifactService.createOrUpdateUnchecked() method that skips validateGroupMembers(). This is cleanest — Bootstrapper calls unchecked for the group, normal path for everything else. Alternative: just catch-and-log ValidationException. Either works; implementer should choose simplest.

Dependency: TASK-0009.01 must produce scripts/agent-lib.sh and scripts/agent-lib-common.sh content before TASK-0009.02 can finalize the seed file bodies. However, seed file bodies can be placeholder text initially and updated once CLI scripts are written. Recommend implementing TASK-0009.01 first.

No unverified assumptions. ApplicationReadyEvent ordering confirmed by Spring Boot lifecycle docs. MetadataCodec.decode() format confirmed by reviewing MetadataCodec.java. ArtifactService.createOrUpdate() API confirmed.

Error handling: Bootstrapper should catch exceptions per-seed-file and log failures without aborting the entire seed process. If one seed fails, others should still be attempted.

Analysis complete. Plan ready. No blockers.
