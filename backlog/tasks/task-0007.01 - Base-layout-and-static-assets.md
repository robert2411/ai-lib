## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Add thymeleaf-layout-dialect dependency to pom.xml (nz.net.ultraq.thymeleaf:thymeleaf-layout-dialect) for decorator-pattern layouts
2. Create src/main/resources/static/css/app.css — minimal responsive styles: nav bar, card grid, flash messages, form styling, utility classes
3. Download htmx.min.js (v1.9.x or 2.0.x) to src/main/resources/static/js/htmx.min.js as a vendored copy (no CDN dependency at runtime)
4. Create src/main/resources/templates/layout.html — Thymeleaf layout decorator template containing:
   - HTML5 doctype, charset, viewport meta
   - <link> to /css/app.css
   - <script> for /js/htmx.min.js with hx-boost="true" on <body>
   - <nav> with: brand/home link, Browse link, New link (th:if EDITOR role), Admin link (th:if ADMIN role), username display, logout form (POST /logout)
   - Flash message block: iterate th:each over "success"/"error" flash attributes, render styled alert divs
   - layout:fragment="content" placeholder for child page content
   - <script> block with layout:fragment="scripts" for page-specific JS (Monaco)
5. Create src/main/resources/templates/fragments/header.html — th:fragment="header" with page title and optional subtitle (used by child pages via th:replace)
6. Create src/main/resources/templates/fragments/artifact-card.html — th:fragment="artifactCard(artifact)" showing: title, type badge, version, description truncated, harness icons, tag chips, link to /artifacts/{slug}
   - For agent-group type: show member count role badge (e.g. "3 agents")
7. Create src/main/resources/templates/fragments/filters.html — th:fragment="filters(types, harnesses, tags, selectedType, selectedHarness, selectedTag)" with checkbox/select inputs that have hx-get, hx-target, hx-include for HTMX partial swap
8. Create src/main/resources/templates/fragments/pagination.html — th:fragment="pagination(currentPage, totalPages, baseUrl)" with prev/next/page-number links preserving query params
9. Create src/main/resources/templates/login.html — standalone page (uses layout.html decorator) with:
   - Form POST /login with username + password fields
   - Error message display (param.error, param.logout)
   - No nav bar (or minimal branding-only nav)
10. Add Monaco editor vendor bundle: create src/main/resources/static/js/monaco/ directory. Document in a README that Monaco files should be fetched from CDN (vs.loader.js and editor worker). Alternatively, reference Monaco from CDN in the scripts fragment (simpler for v1). Decision: use CDN links for Monaco on edit pages only — no build step.
11. Update SecurityConfig: ensure /css/**, /js/**, /login are in permitAll() so static assets load on the login page without auth
12. Test approach:
    - Create WebMvcTest for a minimal controller that renders layout.html — verify 200, presence of nav, htmx script tag
    - Test login.html renders via GET /login with MockMvc — verify form action, CSRF (if re-enabled), field names
    - Verify static resources resolve (MockMvc GET /css/app.css → 200, GET /js/htmx.min.js → 200)
13. Dependencies: None — this is the foundational task. All other tasks in this milestone depend on layout.html and fragments.
