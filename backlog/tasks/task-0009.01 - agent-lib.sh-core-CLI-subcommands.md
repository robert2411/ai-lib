## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
agent-lib.sh bash script with subcommands: install <slug> [--harness] [--version] [--server], search <query> [--type] [--harness], list [--type] [--harness], pull <slug> (download raw file), push <file> (upload artifact), info <slug>. Reads AGENT_LIB_SERVER env var. install accepts agent-group slugs and installs all member agents in role order (manager, analyse, implementation) in a single operation, showing progress per member.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create scripts/agent-lib.sh — Main CLI entrypoint (Bash 3.2+)
   - Shebang: #!/usr/bin/env bash; set -euo pipefail
   - Source agent-lib-common.sh from same directory (dirname $0)
   - Parse AGENT_LIB_SERVER from env (required; error if unset)
   - Parse first positional arg as subcommand; dispatch to function
   - Subcommands: install, search, list, pull, push, info
   - Each subcommand function handles its own --help flag
   - Global --help prints full usage summary

2. Create scripts/agent-lib-common.sh — Shared helper functions
   - _require_cmd(): assert curl, jq, and unzip are available, fatal if missing
   - _api_get(): curl -sSf -u "$AGENT_LIB_USER:$AGENT_LIB_PASS" "$AGENT_LIB_SERVER/api/v1/$1"
   - _api_post(): same with -X POST -d @- or -d "$body"
   - _api_post_file(): curl -sSf -u ... -F "file=@$1" for push
   - _die(): echo error to stderr, exit 1
   - _info(): echo to stderr (progress messages)
   - _resolve_harness(): detect harness from env or default to "claude"
   - Auth: read AGENT_LIB_USER + AGENT_LIB_PASS from env vars only (no config file for MVP)
   - Runtime prerequisites: curl, jq, unzip (checked at startup via _require_cmd)
   - Fallback for unzip: if unzip not found, try python3 -m zipfile; fail if neither available

3. Implement "search" subcommand
   - Usage: agent-lib search <query> [--type T] [--harness H]
   - Calls GET /api/v1/artifacts?q=<query>&type=T&harness=H
   - Formats response as columnar table: SLUG | TYPE | DESCRIPTION (truncated)
   - Uses jq to parse JSON array; prints each artifact on one line
   - Maps to AC #2

4. Implement "list" subcommand
   - Usage: agent-lib list [--type T] [--harness H]
   - Calls GET /api/v1/artifacts (with optional ?type=&harness= params)
   - Formats response as table: SLUG | TYPE | HARNESSES
   - Pagination: loops fetching page=0,1,... until results < size
   - Maps to AC #3

5. Implement "info" subcommand
   - Usage: agent-lib info <slug>
   - Calls GET /api/v1/artifacts/<slug>/latest
   - Pretty-prints all metadata fields: name, type, version, description, harnesses, tags, author
   - Shows install target path for each supported harness

6. Implement "install" subcommand — single artifact
   - Usage: agent-lib install <slug> [--harness H] [--version V] [--server URL] [--dry-run] [--fetch-only] [--dest DIR]
   - Step A: GET /api/v1/install/manifest?slugs=<slug>&harness=<H>
   - Step B: GET /api/v1/install/bundle?slugs=<slug> (downloads zip to temp dir via mktemp -d)
   - Step C: Extract zip using _extract_zip helper:
     * Try: unzip -o "$zipfile" -d "$tmpdir/extracted"
     * Fallback: python3 -m zipfile -e "$zipfile" "$tmpdir/extracted"
     * Fatal if neither available (caught at startup in _require_cmd)
   - Step D: For each entry in manifest.entries:
     * Resolve targetPath (expand ~ to $HOME)
     * If targetPath contains "#" (JSON merge target): invoke _json_merge helper (see Step 6E)
     * Otherwise: mkdir -p parent; cp extracted file to targetPath
   - Step E: _json_merge helper for #-paths (e.g. ~/.cursor/settings.json#mcpServers):
     * Parse target: FILE_PATH="${targetPath%%#*}" KEY_PATH="${targetPath##*#}"
     * If FILE_PATH does not exist: create it with jq -n --argjson val "$(cat $srcfile)" '{"'$KEY_PATH'": $val}' > "$FILE_PATH"
     * If FILE_PATH exists: merge with jq --argjson val "$(cat $srcfile)" '.["'$KEY_PATH'"] += $val' "$FILE_PATH" > "$FILE_PATH.tmp" && mv "$FILE_PATH.tmp" "$FILE_PATH"
     * Validate result is valid JSON (jq . "$FILE_PATH" > /dev/null) or rollback
   - Step F: Print summary per file placed
   - --fetch-only: skip placement; extract zip to --dest (default ./artifact/)
   - --dry-run: print planned actions, do not write
   - Cleanup: rm -rf "$tmpdir" in trap EXIT
   - Maps to AC #1

7. Add "role" field to InstallEntry record (server-side Java change)
   - File: src/main/java/com/agentlibrary/install/InstallEntry.java (or wherever the record lives)
   - Add: String role (nullable) as a new field in the record
   - For non-group single-artifact installs, role will be null
   - For group member entries, role comes from ResolvedGroupMember.role()
   - Update InstallManifestResolver: when building InstallEntry from a ResolvedGroupMember,
     pass member.role() to the new role parameter in the InstallEntry constructor
   - This is a 1-line record change + a small resolver constructor call update
   - Result: manifest JSON now includes "role": "manager" | "analyse" | "implementation" | null
   - Maps to AC #7 (required for CLI to label each member with its role)

8. Implement "install" subcommand — agent-group support (CLI side)
   - Group resolution uses existing manifest API behaviour: GET /api/v1/install/manifest?slugs=<group-slug>&harness=H
   - The manifest endpoint resolves group members and returns expanded entries WITH role field (from Step 7)
   - CLI detects group install when manifest entries contain non-null role fields
   - For each member entry: print "[role] Installing <slug> -> <targetPath>"
   - Progress labels each member with its role (read from manifest entry role field)
   - --dry-run lists all members that would be installed without writing
   - Maps to AC #6, #7, #8

9. Implement "push" subcommand
   - Usage: agent-lib push <file>
   - Reads local file; sends as POST /api/v1/artifacts with Content-Type: text/plain
   - File must contain valid YAML frontmatter
   - Prints confirmation: "Uploaded <slug> v<version>"
   - Maps to AC #4

10. Implement "pull" subcommand
    - Usage: agent-lib pull <slug> [--version V]
    - Calls GET /api/v1/artifacts/<slug>/<version> (or latest)
    - Writes raw content (frontmatter + body) to ./<primaryFileName>
    - Prints filename written

11. Implement --help for all subcommands
    - Each function checks $1 == "--help" || $1 == "-h"
    - Prints usage block with examples
    - Global help: prints all subcommands with one-line description
    - Maps to AC #5

12. Testing strategy — ShellCheck + self-test
    - ShellCheck validation: all scripts pass shellcheck without warnings
    - Add Maven exec-maven-plugin configuration in pom.xml (verify phase):
      * Runs: shellcheck scripts/agent-lib.sh scripts/agent-lib-common.sh
      * Fails build if shellcheck reports any issues
      * Prerequisite: shellcheck installed (brew install shellcheck / apt-get install shellcheck)
    - Script self-test mode: agent-lib --self-test
      * Verifies curl, jq, unzip available
      * Verifies AGENT_LIB_SERVER is reachable (GET /api/v1/health)
      * Dry-run exercises of arg parsing for each subcommand
    - Full integration tests deferred to M11
    - Ensure scripts work with Bash 3.2 (macOS default) — no associative arrays, no |& etc.
