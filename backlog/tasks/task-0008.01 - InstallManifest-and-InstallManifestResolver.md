## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
InstallManifest record: list of InstallEntry (source path in repo, targetPath resolved per harness). InstallManifestResolver resolves target paths for each artifact type + harness combination using a harness path config map. For agent-group slugs: resolver expands the group into its member agents and returns one InstallEntry per member, preserving role order (manager, analyse, implementation).
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Resolver returns correct target path for each ArtifactType x Harness combination
- [ ] #2 Unknown harness throws descriptive exception
- [ ] #3 Manifest serialises to JSON for API response
- [ ] #4 Passing an agent-group slug expands to one entry per member agent
- [ ] #5 Expanded group entries are ordered: manager, analyse, implementation, custom
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create InstallEntry record in com.agentlibrary.install package:
   - Fields: String slug, ArtifactType type, String sourcePath (relative path in repo, e.g. "skills/git-helper/skill.md"), String targetPath (resolved per harness)
   - Immutable Java record with null validation on all fields

2. Create InstallManifest record in com.agentlibrary.install package:
   - Fields: List<InstallEntry> entries, Harness harness
   - Record is serialisable to JSON via Jackson (Spring Boot auto-config handles this)
   - AC#3: The record structure IS the JSON shape — no separate DTO needed since records serialise naturally

3. Create InstallManifestResolver class in com.agentlibrary.install package (Spring @Component):
   - Constructor-injected with ArtifactService (for resolving group members and getting artifact metadata)
   - Define HARNESS_TARGET_PATHS static map<Harness, Map<ArtifactType, String>> with path templates:
     * CLAUDE + SKILL → "~/.claude/skills/{name}/"
     * CLAUDE + AGENT_CLAUDE → ".claude/agents/{name}.md"
     * CLAUDE + MCP → ".claude/settings.json#mcpServers.{name}"
     * CLAUDE + COMMAND → "~/.claude/commands/{name}.md"
     * CLAUDE + HOOK → ".claude/settings.json#hooks.{event}"
     * CLAUDE + CLAUDE_MD → "CLAUDE.md"
     * CLAUDE + PROMPT → "~/.claude/prompts/{name}.md"
     * COPILOT + AGENT_COPILOT → ".github/agents/{name}.md"
     * COPILOT + MCP → ".vscode/mcp.json#servers.{name}"
     * (other combinations as appropriate)
   - Method: InstallManifest resolve(List<String> slugs, Harness harness)
     * For each slug, call ArtifactService.get(slug, "latest") to retrieve metadata
     * If artifact type is AGENT_GROUP: expand via ArtifactService.resolveGroup(slug) → one InstallEntry per member (AC#4)
     * For non-group artifacts: resolve targetPath by looking up (harness, type) in HARNESS_TARGET_PATHS, substituting {name} with artifact name
     * AC#2: If harness has no mapping for the given type, throw IllegalArgumentException with message "No install path for <type> on harness <harness>"
     * AC#5: Group expansion preserves role order from ArtifactService.resolveGroup() which already sorts: manager=0, analyse=1, implementation=2, custom=3
   - Private method: String resolveTargetPath(ArtifactMetadata metadata, Harness harness)
     * Looks up harness path config map
     * Substitutes {name} placeholder with metadata.name()
     * Returns resolved path string

4. Create InstallManifestResolverTest (unit test, Mockito):

   - Test: resolve agent-claude for claude → .claude/agents/{name}.md (AC#1)

   - Test: resolve agent-group slug → expand to member entries, verify order manager→analyse→implementation→custom (AC#4, AC#5)
   - Test: resolve agent-group with mixed roles → correct ordering (AC#5)
   - Test: JSON serialisation of InstallManifest (AC#3) — serialise via ObjectMapper, verify JSON structure has "entries" array with slug/type/sourcePath/targetPath and "harness" field
