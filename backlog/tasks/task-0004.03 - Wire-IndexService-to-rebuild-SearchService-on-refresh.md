## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
IndexService.refresh() calls SearchService.rebuild(allArtifacts) after updating the in-memory cache. FacetService extracts distinct tags, types, harnesses from index for filter dropdown population.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 After repo save, SearchService reflects new artifact without restart
- [ ] #2 FacetService returns correct distinct tag/type/harness lists
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Modify `IndexService.refresh()` (from TASK-0004.01) to call `SearchService.rebuild()` after cache update:
   - Inject `SearchService` into `IndexService` constructor.
   - In `refresh()`, after updating the cache, call `searchService.rebuild(cache)`.
   - Also call `searchService.rebuild(cache)` in `init()` after initial load.
   - This ensures SearchService is always in sync with the index.
   - AC #1: After repo save, SearchService reflects new artifact without restart.

2. Wire repository save/delete to trigger `IndexService.refresh()`:
   - Option A (preferred): Add a callback/listener mechanism.
     - Define interface `IndexRefreshListener` with method `void onRefresh()`.
     - Or simpler: inject `IndexService` into a post-commit hook.
   - Option B (simplest, recommended): Have `LocalGitArtifactRepository` accept an optional `Runnable` or listener called after commit.
   - Option C (event-based): Use Spring `ApplicationEventPublisher` to publish a `CommitEvent` after save/delete. `IndexService` listens with `@EventListener`.
   - **Recommended: Option C (Spring Events)** — cleanest decoupling:
     a. Create `ArtifactCommitEvent extends ApplicationEvent` in `com.agentlibrary.index`.
     b. Inject `ApplicationEventPublisher` into `LocalGitArtifactRepository`.
     c. After `save()` and `delete()` succeed, publish `ArtifactCommitEvent`.
     d. `IndexService` has `@EventListener` method that calls `refresh()`.
   - Alternative (if simpler wiring preferred): Just inject IndexService directly into the repository and call refresh(). This creates a dependency from storage→index, which is acceptable for now.
   - **Final decision for implementation: Direct injection.** The repository calls `indexService.refresh()` after save/delete commits. This is explicit, testable, and avoids over-engineering for a single-process app.
   - Update `StorageConfig` to wire the dependency, or use setter/optional injection to avoid circular deps.

3. Handle circular dependency risk:
   - IndexService depends on ArtifactRepository (to call list()).
   - If ArtifactRepository depends on IndexService (to call refresh()), this is circular.
   - Solution: Use `@Lazy` on one injection, OR use an event-based approach, OR use a `RefreshCallback` functional interface:
     a. Create `RefreshCallback` interface: `@FunctionalInterface interface RefreshCallback { void onCommit(); }`
     b. `LocalGitArtifactRepository` accepts `Optional<RefreshCallback>` or setter `setRefreshCallback(RefreshCallback cb)`.
     c. Config wires: `repository.setRefreshCallback(() -> indexService.refresh())`.
   - **Recommended: setter-based callback.** Add `public void setRefreshCallback(Runnable callback)` to `LocalGitArtifactRepository`. `IndexConfig` wires it after both beans are created using `@PostConstruct` or a `@Bean` method.

4. Create `FacetService.java` in `com.agentlibrary.index`:
   - Spring `@Service`, injected with `IndexService`.
   - Computes distinct values from the cached metadata list.
   - Methods:
     - `public List<String> getDistinctTags()` — collect all unique tags from all artifacts, sorted.
     - `public List<ArtifactType> getDistinctTypes()` — collect all unique types present.
     - `public List<Harness> getDistinctHarnesses()` — collect all unique harnesses present.
     - `public List<String> getDistinctCategories()` — collect all unique categories.
     - `public List<String> getDistinctAuthors()` — collect all unique authors.
   - Implementation: iterate `indexService.getAll()`, collect into TreeSet/LinkedHashSet.
   - These are computed on-demand (not cached) since getAll() is O(1) from cache and <10k items.
   - AC #2: FacetService returns correct distinct tag/type/harness lists.

5. Create `IndexConfig.java` in `com.agentlibrary.index`:
   - `@Configuration` class.
   - Wire the refresh callback:
     ```java
     @Bean
     public CommandLineRunner wireRefreshCallback(LocalGitArtifactRepository repo, IndexService indexService) {
         return args -> repo.setRefreshCallback(indexService::refresh);
     }
     ```
   - Or use a `@PostConstruct` in a config bean.

6. Modify `LocalGitArtifactRepository`:
   - Add field: `private Runnable refreshCallback;`
   - Add setter: `public void setRefreshCallback(Runnable callback) { this.refreshCallback = callback; }`
   - At end of `doSave()` (after updateIndex), call: `if (refreshCallback != null) refreshCallback.run();`
   - At end of `doDelete()` (after updateIndex), call: `if (refreshCallback != null) refreshCallback.run();`
   - This is the minimal change to the existing storage layer.

7. Create tests:
   a. `src/test/java/com/agentlibrary/index/FacetServiceTest.java`:
      - Mock IndexService to return known metadata list.
      - Test: getDistinctTags() returns all unique tags sorted.
      - Test: getDistinctTypes() returns only types present in data.
      - Test: getDistinctHarnesses() returns only harnesses present.
      - Test: empty index returns empty lists.
   b. Integration test `src/test/java/com/agentlibrary/index/IndexRefreshIntegrationTest.java`:
      - Uses real IndexService + SearchService + FacetService.
      - Mocked repository that returns different data on each list() call.
      - Test: after refresh(), SearchService.search() finds newly added artifact (AC #1).
      - Test: after refresh(), FacetService reflects new tags (AC #2).
   c. Modify existing `LocalGitArtifactRepositorySaveTest` (or add new test) to verify callback is invoked.

8. Files to create:
