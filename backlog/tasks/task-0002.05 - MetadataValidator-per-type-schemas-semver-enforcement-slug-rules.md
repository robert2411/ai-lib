## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
MetadataValidator validates ArtifactMetadata against per-type YAML schemas loaded from TypeSchemaRegistry. Enforces slug format (lowercase, hyphens), valid SemVer version, required fields per type, harness allowed values per type. Includes schema/agent-group.yaml defining the members[] structure with required slug and role fields.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Validation passes for fixture artifact of each ArtifactType
- [ ] #2 Missing required field returns descriptive ValidationException
- [ ] #3 Invalid slug format is rejected
- [ ] #4 Invalid semver is rejected
- [ ] #5 agent-group type requires at least one member with a recognised role
- [ ] #6 agent-group with duplicate member slugs is rejected
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Depends on: TASK-0002.01 (enums), TASK-0002.02 (ArtifactMetadata), TASK-0002.03 (SemVer), TASK-0002.04 (MetadataCodec)
2. Create class ValidationException in com.agentlibrary.metadata.ValidationException:
   - File: src/main/java/com/agentlibrary/metadata/ValidationException.java
   - Extends RuntimeException
   - Field: List<String> errors (unmodifiable)
   - Constructor: ValidationException(List<String> errors)
   - Method: public List<String> errors()
   - Override getMessage() to join all errors with "; "
3. Create class TypeSchemaRegistry in com.agentlibrary.metadata.TypeSchemaRegistry:
   - File: src/main/java/com/agentlibrary/metadata/TypeSchemaRegistry.java
   - Loads per-type schema definitions from classpath YAML files
   - Schema location: src/main/resources/schemas/<type-slug>.yaml (e.g. schemas/skill.yaml, schemas/agent-group.yaml)
   - Each schema YAML defines:
     - required: [list of required field names]
     - optional: [list of optional field names]
     - allowedHarnesses: [list of harness slugs] (or "all")
   - Method: public static TypeSchema forType(ArtifactType type)
   - Fallback: if no schema file exists for a type, use a default schema with minimal required fields (name, type, version)
4. Create record TypeSchema in com.agentlibrary.metadata.TypeSchema:
   - File: src/main/java/com/agentlibrary/metadata/TypeSchema.java
   - Fields: Set<String> requiredFields, Set<String> optionalFields, Set<Harness> allowedHarnesses
5. Create schema YAML files:
   - src/main/resources/schemas/skill.yaml: required: [name, type, version, description], allowedHarnesses: [claude, copilot]
   - src/main/resources/schemas/agent-claude.yaml: required: [name, type, version], allowedHarnesses: [claude]
   - src/main/resources/schemas/agent-copilot.yaml: required: [name, type, version], allowedHarnesses: [copilot]
   - src/main/resources/schemas/agent-group.yaml: required: [name, type, version, members], allowedHarnesses: all, members schema: each member requires slug and role
   - src/main/resources/schemas/mcp.yaml: required: [name, type, version], allowedHarnesses: all
   - src/main/resources/schemas/command.yaml: required: [name, type, version], allowedHarnesses: [claude]
   - src/main/resources/schemas/hook.yaml: required: [name, type, version], allowedHarnesses: [claude]
   - src/main/resources/schemas/claude-md.yaml: required: [name, type, version], allowedHarnesses: [claude]
   - src/main/resources/schemas/opencode.yaml: required: [name, type, version], allowedHarnesses: all
   - src/main/resources/schemas/prompt.yaml: required: [name, type, version], allowedHarnesses: all
6. Create class MetadataValidator in com.agentlibrary.metadata.MetadataValidator:
   - File: src/main/java/com/agentlibrary/metadata/MetadataValidator.java
   - Method: public static void validate(ArtifactMetadata metadata) throws ValidationException
   - Validation steps (collect ALL errors then throw once):
     a. Slug validation: metadata.name() must match ^[a-z][a-z0-9-]*[a-z0-9]$ (or single char ^[a-z]$)
        - Lowercase letters, digits, hyphens only
        - Must start with letter
        - Must not end with hyphen
        - Must not contain consecutive hyphens
     b. SemVer validation: SemVer.isValid(metadata.version()) must be true
     c. Required fields: load TypeSchema for metadata.type(), check all required fields are non-null and non-blank
     d. Harness validation: each harness in metadata.harnesses() must be in schema allowedHarnesses (unless "all")
     e. Agent-group specific:
        - If type == AGENT_GROUP, members list must not be empty (AC #5)
        - Each member must have non-blank slug and a recognised role
        - No duplicate member slugs (AC #6)
     f. Collect all validation errors into a list
     g. If errors non-empty, throw ValidationException(errors)
7. Create test fixtures:
   - src/test/resources/fixtures/ directory
   - One valid fixture YAML per ArtifactType (10 files)
   - e.g. fixtures/valid-skill.yaml, fixtures/valid-agent-group.yaml, etc.

   - Test: agentGroupRequiresMembers (AC #5):

   - Test: agentGroupRequiresRecognisedRole (AC #5):
