## Description

AdminController GET /admin, POST /admin/reindex. admin.html: reindex button (HTMX POST, shows success/error toast), read-only users table (username, roles). Requires ROLE_ADMIN.

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create src/main/java/com/agentlibrary/web/AdminController.java in package com.agentlibrary.web:
   - @Controller, @PreAuthorize("hasRole(\"ADMIN\")") at class level
   - Constructor-injected with IndexService and UsersFile
   - @GetMapping("/admin") → returns "admin" view name
   - @PostMapping("/admin/reindex") → triggers reindex, returns HTMX fragment or redirect
2. Implement GET /admin handler:
   - Load users: usersFile.loadAll() → List<UserEntry>
   - Map to view-friendly DTOs (or use UserEntry directly since it has username and roles)
   - Add to model: "users" (list of user entries with username + roles), "indexCount" (indexService.getAll().size())
   - Return "admin"
3. Implement POST /admin/reindex handler:
   - Call indexService.refresh()
   - Detect HTMX request via HX-Request header:
     * If HTMX: return a fragment with success message toast, e.g. "fragments/admin-toast :: toast" with model attribute "toastMessage" = "Index rebuilt — N artifacts", "toastType" = "success"
     * If non-HTMX (fallback): add flash attribute "success" = "Index rebuilt successfully", redirect to /admin
   - Handle exceptions: if refresh throws, set toastType = "error", toastMessage = error details
4. Create src/main/resources/templates/admin.html using layout.html decorator:
   - Page heading: "Administration"
   - Section 1 — Reindex:
     * Current index count display: "Index contains N artifacts"
     * Reindex button: <button hx-post="/admin/reindex" hx-target="#reindex-result" hx-swap="innerHTML">Rebuild Index</button>
     * Result area: <div id="reindex-result"></div> where toast fragment gets swapped in
   - Section 2 — Users:
     * Read-only table with columns: Username, Roles
     * Iterate th:each="user : ${users}" displaying user.username() and user.roles() (joined as comma-separated string)
     * Note: no user management CRUD — read-only display (per task scope)
5. Create src/main/resources/templates/fragments/admin-toast.html:
   - th:fragment="toast" containing a styled alert/toast div
   - Uses th:text="${toastMessage}" and th:classappend for success/error styling
   - Auto-dismiss via CSS animation or small JS setTimeout to fade out
6. Security enforcement:
   - Class-level @PreAuthorize("hasRole(\"ADMIN\")") covers both GET and POST
   - Spring Security will return 403 for non-ADMIN users
   - Since formLogin is configured, non-admin authenticated users hitting /admin get 403 (Access Denied)
   - Consider adding a custom AccessDeniedHandler that redirects to / with flash error (optional enhancement, not required by AC)
7. Test approach — create AdminControllerTest.java:
   - @WebMvcTest(AdminController.class) with @MockBean IndexService, UsersFile, Bucket
   - Test GET /admin with ADMIN role: returns 200, view "admin", model contains "users" list
   - Test GET /admin with USER role: returns 403
   - Test GET /admin with EDITOR role: returns 403 (EDITOR does not inherit ADMIN)
   - Test POST /admin/reindex with ADMIN role + HX-Request header: returns 200 with toast fragment containing success message
   - Test POST /admin/reindex with ADMIN role without HX-Request: redirects to /admin with flash
   - Test POST /admin/reindex with non-ADMIN role: returns 403
   - Mock usersFile.loadAll() to return test entries, verify table content in rendered HTML
8. AC mapping:
   - AC#1 (GET /admin renders users table from UsersFile): loadAll() → model → th:each table
   - AC#2 (reindex button triggers POST and shows confirmation): hx-post + toast fragment swap
   - AC#3 (non-admin redirected away): @PreAuthorize + 403 response
9. Dependencies: Requires TASK-0007.01 (layout.html). Independent of TASK-0007.02/03/04/05.
