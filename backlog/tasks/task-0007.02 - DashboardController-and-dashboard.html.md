## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
DashboardController serves GET /. dashboard.html shows recent artifacts (last 10), total count by type, quick search form. Data provided by IndexService.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 GET / returns 200 with rendered dashboard
- [ ] #2 Recent artifacts list shows up to 10 entries
- [ ] #3 Type counts are accurate
- [ ] #4 Search form submits to /browse with q param
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create src/main/java/com/agentlibrary/web/DashboardController.java in package com.agentlibrary.web:
   - @Controller, constructor-injected with ArtifactService, IndexService, FacetService
   - @GetMapping("/") → returns "dashboard" view name
   - Model attributes: "recentArtifacts" (last 10 by updated/created desc), "typeCounts" (Map<ArtifactType, Long>), "totalCount" (long)
2. Implement recentArtifacts: call indexService.getAll(), sort by metadata.updated() (or created()) descending, take first 10. Return as List<ArtifactMetadata>.
3. Implement typeCounts: call indexService.getAll(), group by ArtifactType using Collectors.groupingBy + Collectors.counting(). Return as Map<ArtifactType, Long> for display.
4. Implement totalCount: indexService.getAll().size() — simple total.
5. Create src/main/resources/templates/dashboard.html using layout.html decorator:
   - layout:decorate="~{layout}" with layout:fragment="content"
   - "Welcome" heading or hero section
   - Quick-search form: <form action="/browse" method="get"> with <input name="q"> and submit button. Uses standard form submission (not HTMX) to navigate to /browse with query param.
   - "Recent Artifacts" section: iterate th:each="artifact : ${recentArtifacts}" using th:replace of fragments/artifact-card.html
   - "By Type" section: iterate over typeCounts map, display type name + count in badge/pill format. Each type links to /browse?type={slug}.
6. Handle empty state: if recentArtifacts is empty, show "No artifacts yet. Create one!" message with link to /new.
7. Test approach — create DashboardControllerTest.java:
   - @WebMvcTest(DashboardController.class) with @MockBean ArtifactService, IndexService, FacetService, Bucket
   - @WithMockUser(roles="USER")
   - Test GET / returns 200 and view name "dashboard"
   - Test model contains "recentArtifacts" (mock indexService.getAll() to return sample list)
   - Test model contains "typeCounts" with correct grouping
   - Test search form is present in HTML (check for action="/browse")
   - Test empty state: when indexService.getAll() returns empty, page still renders 200
8. AC mapping:
   - AC#1 (GET / returns 200): covered by controller + template
   - AC#2 (recent artifacts up to 10): covered by sort+limit logic
   - AC#3 (type counts accurate): covered by groupingBy logic
   - AC#4 (search form submits to /browse with q): covered by form action attribute
9. Dependencies: Requires TASK-0007.01 (layout.html, fragments/artifact-card.html) to be complete for template rendering.
<!-- SECTION:PLAN:END -->
