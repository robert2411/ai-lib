## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
GET /api/v1/install/manifest?slugs=a,b&harness=claude returns InstallManifest JSON. GET /api/v1/install/bundle?slugs=a,b returns zip of artifact files. BundleService wraps ArtifactRepository.bundle().
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 GET manifest returns correct install paths for requested slugs and harness
- [ ] #2 GET bundle returns zip with Content-Disposition header
- [ ] #3 Unknown slug in request returns 404
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create BundleService in com.agentlibrary.install package (Spring @Service):
   - Constructor-injected with ArtifactService
   - Method: byte[] bundleAsZip(List<String> slugs)
     * For each slug, call ArtifactService.get(slug, "latest") to retrieve Artifact (metadata + content)
     * If slug is agent-group type, expand group via ArtifactService.resolveGroup() and bundle each member
     * Build a zip archive (java.util.zip.ZipOutputStream) containing all artifact files
     * Each artifact stored under folder: {type-slug}/{name}/ (e.g. "skill/git-helper/skill.md")
     * Determine primary filename from ArtifactType conventions (skill→skill.md, agent-claude→agent.md, etc.)
     * If any slug is unknown, throw NotFoundException (bubbles from ArtifactService.get())
     * Return byte[] of the complete zip archive
   - NOTE on format: AC#2 says "zip" — this is intentional for the multi-slug install endpoint. The per-artifact /artifacts/{type}/{name}/bundle endpoint stays tar.gz (existing). The install bundle uses zip because CLI consumers (curl + unzip) prefer it.

2. Create InstallApiController in com.agentlibrary.web.api package:
   - @RestController @RequestMapping("/api/v1/install")
   - Constructor-injected with InstallManifestResolver and BundleService

3. GET /api/v1/install/manifest endpoint:
   - Parameters: @RequestParam List<String> slugs, @RequestParam String harness
   - Parse harness via Harness.fromSlug(harness) — unknown harness → IllegalArgumentException → 400 (via ApiExceptionHandler)
   - Call InstallManifestResolver.resolve(slugs, harness)
   - If any slug is unknown → NotFoundException → 404 (AC#3, handled by ApiExceptionHandler)
   - Return ResponseEntity.ok(manifest) — Jackson auto-serialises the InstallManifest record
   - AC#1: verified by testing correct paths in response JSON

4. GET /api/v1/install/bundle endpoint:
   - Parameters: @RequestParam List<String> slugs
   - Call BundleService.bundleAsZip(slugs)
   - If any slug is unknown → NotFoundException → 404 (AC#3)
   - Return ResponseEntity with:
     * Content-Type: application/zip
     * Content-Disposition: attachment; filename="install-bundle.zip" (AC#2)
     * Body: byte[] from BundleService
   - AC#2: verified by checking Content-Disposition header and Content-Type in test

5. Create InstallApiControllerTest (MockMvc unit test):
   - @WebMvcTest(InstallApiController.class)
   - @MockBean InstallManifestResolver, @MockBean BundleService, @MockBean Bucket loginRateLimitBucket
   - Test: GET /manifest with valid slugs+harness → 200 + correct JSON (AC#1)
   - Test: GET /manifest with unknown slug → 404 (AC#3)
   - Test: GET /manifest with unknown harness → 400
   - Test: GET /bundle with valid slugs → 200 + application/zip content type + Content-Disposition header (AC#2)
   - Test: GET /bundle with unknown slug → 404 (AC#3)

6. Create BundleServiceTest (unit test, Mockito):
   - Mock ArtifactService
   - Test: bundleAsZip with single slug → valid zip containing expected file
   - Test: bundleAsZip with multiple slugs → zip contains all artifacts
   - Test: bundleAsZip with unknown slug → NotFoundException propagates
   - Test: bundleAsZip with agent-group slug → zip contains expanded members

7. File locations:
   - src/main/java/com/agentlibrary/install/BundleService.java

   - src/test/java/com/agentlibrary/web/api/InstallApiControllerTest.java
