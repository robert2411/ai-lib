## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
GET /api/v1/install/manifest?slugs=a,b&harness=claude returns InstallManifest JSON. GET /api/v1/install/bundle?slugs=a,b returns zip of artifact files. BundleService wraps ArtifactRepository.bundle().
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 GET manifest returns correct install paths for requested slugs and harness
- [ ] #2 GET bundle returns zip with Content-Disposition header
- [ ] #3 Unknown slug in request returns 404
<!-- AC:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create BundleService in com.agentlibrary.install package (Spring @Service):
   - Constructor-injected with ArtifactService
   - Method: byte[] bundleAsZip(List<String> slugs)
     * Validate slugs list is not empty — throw IllegalArgumentException if empty
     * For each slug, call ArtifactService.bundle(slug, "latest") which returns InputStream of tar.gz
     * If slug is agent-group type: first call ArtifactService.resolveGroup(slug) to get members, then bundle each member individually
     * For each artifact tar.gz stream: extract entries using TarArchiveInputStream+GzipCompressorInputStream
     * Re-package all extracted files into a single zip (java.util.zip.ZipOutputStream)
     * Zip structure: {slug}/ prefix per artifact (e.g. "git-helper/skill.md", "my-agent/agent.md")
     * If any slug is unknown, NotFoundException propagates from ArtifactService
     * Return byte[] of the complete zip archive
   - NOTE on format: AC#2 says "zip" — this is intentional for the multi-slug install endpoint. The existing per-artifact /artifacts/{type}/{name}/bundle remains tar.gz. The install bundle converts to zip because the install CLI uses `unzip` which is more portable.
   - Implementation detail: Uses existing ArtifactService.bundle() which returns tar.gz per slug → extract → re-package as zip. This "wraps ArtifactRepository.bundle()" per task description.

2. Create InstallApiController in com.agentlibrary.web.api package:
   - @RestController @RequestMapping("/api/v1/install")
   - Constructor-injected with InstallManifestResolver and BundleService

3. GET /api/v1/install/manifest endpoint:
   - @GetMapping("/manifest")
   - Parameters: @RequestParam List<String> slugs, @RequestParam String harness
   - Parse harness via Harness.fromSlug(harness) — unknown harness → IllegalArgumentException → 400 (via existing ApiExceptionHandler)
   - Call InstallManifestResolver.resolve(slugs, parsedHarness)
   - If any slug is unknown → NotFoundException → 404 (AC#3, handled by existing ApiExceptionHandler)
   - Return ResponseEntity.ok(manifest) — Jackson auto-serialises the InstallManifest record
   - AC#1: verified by testing correct paths in response JSON

4. GET /api/v1/install/bundle endpoint:
   - @GetMapping("/bundle")
   - Parameters: @RequestParam List<String> slugs
   - Call BundleService.bundleAsZip(slugs)
   - If any slug is unknown → NotFoundException → 404 (AC#3)
   - Return ResponseEntity with:
     * Content-Type: application/zip (MediaType "application/zip")
     * Content-Disposition: attachment; filename="install-bundle.zip" (AC#2)
     * Body: byte[] from BundleService
   - Use ResponseEntity.ok().header(HttpHeaders.CONTENT_DISPOSITION, ...).contentType(...).body(bytes)
   - AC#2: verified by checking Content-Disposition header and Content-Type in test

5. Create InstallApiControllerTest (MockMvc unit test):

   - @MockBean InstallManifestResolver, @MockBean BundleService, @MockBean Bucket loginRateLimitBucket

   - Test: GET /api/v1/install/manifest?slugs=unknown&harness=claude → 404 (AC#3)
   - Test: GET /api/v1/install/manifest?slugs=git-helper&harness=badharness → 400 (unknown harness)

   - Test: GET /api/v1/install/bundle?slugs=unknown → 404 (AC#3)
   - Test: unauthenticated request → 401 (all endpoints require auth per design doc)
