## Implementation Notes

<!-- SECTION:NOTES:BEGIN -->
Self-review findings (addressed in plan):

1. Rate limit endpoint: The RateLimitFilter intercepts only POST /login (form login path). The E2E test must POST to /login with form data (username=admin&password=wrong), NOT Basic Auth headers. The filter checks request URI "/login" + method POST. Assert first 10 → 302 redirect or 401, 11th → 429.

2. CRUD Content-Type: POST /api/v1/artifacts consumes "text/plain" or "application/json". The test should send raw markdown with Content-Type: text/plain and Authorization: Basic header.

3. Build lifecycle: ImageFromDockerfile builds during integration-test phase. Since maven-failsafe runs AFTER package phase, the JAR exists in target/. The Dockerfile COPYs from ./target/agent-library-*.jar via the build stage, so ImageFromDockerfile with full project context will work. However, building the full image each test run is slow (~60s). Alternative: Tag image during package phase with docker-maven-plugin. Recommend ImageFromDockerfile for self-containment; note it adds build time.

4. Docker skip: Use @BeforeAll with Assumptions.assumeTrue(DockerClientFactory.instance().isDockerAvailable()) at class level to skip entire test class gracefully when Docker is absent. This ensures ./mvnw verify always succeeds (tests skipped, not failed).
