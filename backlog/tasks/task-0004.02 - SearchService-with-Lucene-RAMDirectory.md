## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
SearchService builds a Lucene RAMDirectory index from artifact content + metadata. Exposes search(query, Filter) returning ranked List<ArtifactMetadata>. Rebuilt from scratch on each IndexService refresh.
<!-- SECTION:DESCRIPTION:END -->

2. Fields:

3. Lucene version note: Using Lucene 9.11.1. RAMDirectory is deprecated â€” use `org.apache.lucene.store.ByteBuffersDirectory` instead (in-memory, same semantics, not deprecated).

4. Implement `public void rebuild(List<ArtifactMetadata> artifacts)`:
   a. Create a new `ByteBuffersDirectory`.
   b. Create `IndexWriterConfig` with `StandardAnalyzer` and `OpenMode.CREATE`.
   c. Open `IndexWriter` on the new directory.
   d. For each `ArtifactMetadata` in artifacts, create a Lucene `Document`:
      - `StringField("id", type.slug() + "/" + name, Field.Store.YES)` â€” exact match, stored for retrieval.
      - `TextField("title", meta.title() != null ? meta.title() : "", Field.Store.NO)` â€” full-text searchable.
      - `TextField("description", meta.description() != null ? meta.description() : "", Field.Store.NO)` â€” full-text searchable.
      - `TextField("all", combinedText, Field.Store.NO)` â€” concatenation of title + description for default search field.
      - `StringField("type", meta.type().slug(), Field.Store.YES)` â€” for faceted filtering (exact match).
      - For each harness: `StringField("harness", h.slug(), Field.Store.YES)` â€” multi-valued for filtering.
      - For each tag: `StringField("tag", tag, Field.Store.YES)` â€” multi-valued for filtering.
      - `StringField("category", meta.category() != null ? meta.category() : "", Field.Store.YES)` â€” for filtering.
      - `StringField("author", meta.author() != null ? meta.author() : "", Field.Store.YES)` â€” for filtering.
      - `StringField("language", meta.language() != null ? meta.language() : "", Field.Store.YES)` â€” for filtering.
   e. `writer.commit()` and `writer.close()`.
   f. Create new `DirectoryReader` and `IndexSearcher` from the new directory.
   g. Build new `metadataLookup` map.
   h. Acquire write lock, swap `searcher`, `directory`, and `metadataLookup`, release lock.
   i. Close old directory (if any) after swap.
   - AC #4: Rebuilt index after refresh reflects newly saved artifacts.

5. Implement `public List<ArtifactMetadata> search(String query, Filter filter)`:
   a. Acquire read lock.
   b. If searcher is null (not yet built), return empty list.
   c. Build Lucene query:
      - If `query` is non-null and non-blank:
        - Use `QueryParser` on field "all" with `StandardAnalyzer` to parse the text query.
        - Wrap in BooleanQuery with MUST.
      - If filter.type() != null: add `TermQuery("type", filter.type().slug())` as MUST clause.
      - If filter.harness() != null: add `TermQuery("harness", filter.harness().slug())` as MUST clause.
      - If filter.tag() != null: add `TermQuery("tag", filter.tag())` as MUST clause.
      - If all clauses empty (no query, no filter): use `MatchAllDocsQuery`.
   d. Execute search with `searcher.search(luceneQuery, maxResults)` where maxResults = 1000 (sufficient for <10k artifacts).
   e. For each `ScoreDoc` in results, retrieve stored "id" field, look up in `metadataLookup`.
   f. Return ranked `List<ArtifactMetadata>`.
   g. Release read lock in finally block.
   - AC #1: search returns relevant results for keyword query.
   - AC #2: search respects type/harness/tag filter alongside text query.
   - AC #3: Empty query with filter returns all matching artifacts (MatchAllDocsQuery + filter terms).

6. Error handling:
   - If `QueryParser.parse()` throws `ParseException`, catch and fall back to a simple `TermQuery` or return empty list with a logged warning.
   - If searcher is null (before first rebuild), return `List.of()`.
