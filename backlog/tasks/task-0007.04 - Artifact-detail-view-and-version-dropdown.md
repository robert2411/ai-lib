## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
ArtifactViewController GET /artifacts/{slug} (redirect to latest) and /artifacts/{slug}/{version}. artifact.html: metadata panel, rendered content (Markdown via marked.js or pre), version dropdown using HTMX to swap content area. For agent-group type: renders a members panel listing each agent with its role badge (manager, analyse, implementation), links to individual agent pages, and a single-click Install Group button.
<!-- SECTION:DESCRIPTION:END -->

## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create src/main/java/com/agentlibrary/web/ArtifactViewController.java in package com.agentlibrary.web:
   - @Controller, constructor-injected with ArtifactService
   - @GetMapping("/artifacts/{slug}") → redirect to /artifacts/{slug}/latest (302)
   - @GetMapping("/artifacts/{slug}/{version}") → returns "artifact" view name
   - @GetMapping("/artifacts/{slug}/{version}/content") → returns "fragments/artifact-content :: content" for HTMX version swap
   - @GetMapping("/artifacts/{slug}/{version}/raw") → returns raw content as text/plain ResponseEntity for download
2. Implement GET /artifacts/{slug} handler:
   - Fetch latest version: artifactService.get(slug, "latest")
   - Redirect to /artifacts/{slug}/{latestVersion} using RedirectView or "redirect:" prefix
3. Implement GET /artifacts/{slug}/{version} handler:
   - Fetch artifact: artifactService.get(slug, version)
   - Fetch all versions: artifactService.getVersions(slug) — for dropdown
   - Add to model: "artifact" (Artifact), "versions" (List<VersionRef>), "currentVersion" (String)
   - Detect agent-group type: if artifact.metadata().type() == AGENT_GROUP, resolve members via artifactService.resolveGroup(slug) and add "members" (List<ResolvedGroupMember>) to model
   - Add "installCommand" to model: formatted CLI command string e.g. "agent-lib install {type}/{slug}@{version}"
   - Return "artifact"
4. Implement GET /artifacts/{slug}/{version}/content handler:
   - For HTMX version dropdown swap
   - Fetch artifact at specified version: artifactService.get(slug, version)
   - If agent-group: resolve members and add to model
   - Return "fragments/artifact-content :: content"
5. Implement GET /artifacts/{slug}/{version}/raw handler:
   - Fetch artifact content, return ResponseEntity<String> with Content-Type text/plain and Content-Disposition attachment header
6. Create src/main/resources/templates/artifact.html using layout.html decorator:
   - Header: artifact title, type badge, version badge, author
   - Metadata panel: description, harnesses, tags, category, created/updated dates
   - Version dropdown: <select> with hx-get that swaps content area
     * Options iterate over versions list, selected matches currentVersion
     * hx-get="/artifacts/{slug}/{selectedVersion}/content" hx-target="#content-area" hx-trigger="change" (use JS to build URL from select value, or use hx-vals)
     * Better approach: use hx-get with a template URL and swap the value via JS event handler, OR use a form with hx-get
   - Content area div id="content-area":
     * For markdown content: render as <pre><code> block (or use marked.js client-side rendering if available)
     * For non-markdown: render in <pre> block
   - Action buttons row:
     * "Raw Download" link → /artifacts/{slug}/{version}/raw
     * "Copy Install Command" button with data attribute holding the command, JS onclick copies to clipboard
     * "Edit" link (th:if has EDITOR role) → /edit/{slug}/{version}
   - Agent-group members panel (th:if="${artifact.metadata().type().name() == 'AGENT_GROUP'}"): 
     * Heading "Group Members"
     * List/table of members: each shows slug (linked to /artifacts/{memberSlug}), role badge (styled: manager=blue, analyse=green, implementation=purple)
     * "Install Group" button that copies the group install CLI command to clipboard
7. Create src/main/resources/templates/fragments/artifact-content.html:
   - th:fragment="content" containing the content display area (same markup as in artifact.html content-area) so HTMX can swap just the content when version changes
8. Add clipboard-copy JavaScript:
   - Small inline script or utility in app.js: function copyToClipboard(text) using navigator.clipboard.writeText()
   - Attach to buttons via onclick or data-copy attribute
9. Test approach — create ArtifactViewControllerTest.java:
   - @WebMvcTest(ArtifactViewController.class) with @MockBean ArtifactService, Bucket
   - Test GET /artifacts/{slug} redirects to /artifacts/{slug}/{latestVersion}
   - Test GET /artifacts/{slug}/{version} returns 200 with model containing artifact and versions
   - Test GET /artifacts/{slug}/{version}/content with HX-Request returns fragment
   - Test GET /artifacts/{slug}/{version}/raw returns text/plain content
   - Test agent-group detail includes members in model
   - Test copy-install-command data attribute present in rendered HTML
   - Test 404 handling when slug not found (NotFoundException → appropriate error)
10. AC mapping:
    - AC#1 (redirect to latest): GET /artifacts/{slug} → 302 to latest version URL
    - AC#2 (version dropdown swaps content): hx-get on select change targeting #content-area
    - AC#3 (raw content download): /raw endpoint with text/plain response
    - AC#4 (copy install command button): clipboard JS + displayed command
    - AC#5 (agent-group members panel with role badges): conditional members section in artifact.html
    - AC#6 (member links to own detail page): each member slug linked to /artifacts/{memberSlug}
    - AC#7 (Install Group button copies CLI command): button with data-copy="agent-lib install agent-group/{slug}"
11. Dependencies: Requires TASK-0007.01 (layout.html). Independent of TASK-0007.02/03.
