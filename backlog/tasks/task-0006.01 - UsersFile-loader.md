## Implementation Plan — TASK-0006.01: UsersFile Loader

### Context
SecurityConfig already contains a private `loadUsersFromYaml(Path)` method that parses users.yaml and builds Spring Security UserDetails. This task extracts that YAML-loading logic into a dedicated, testable class in the `com.agentlibrary.auth` package, returning domain-level User records (not Spring UserDetails).

### Steps

1. **Create `com.agentlibrary.auth.UsersFile` class**
   - File: `src/main/java/com/agentlibrary/auth/UsersFile.java`
   - Constructor takes `Path usersFilePath` (injected via AppProperties)
   - Make it a `@Component` with constructor injection of `AppProperties`
   - Add `@PostConstruct` or eager load in constructor
   - Internal record `UserEntry(String username, String encodedPassword, List<String> roles)`
   - Method: `List<UserEntry> loadAll()` — parses YAML, returns user entries

2. **Implement YAML parsing in loadAll()**
   - Use SnakeYAML `SafeConstructor` (already a dependency)
   - Open file from `props.getUsersFile()` path
   - Parse structure: `users: [{username, password, roles: [...]}]`
   - Validate all required fields present (username, password, roles)
   - Return immutable `List<UserEntry>`

3. **Error handling**
   - If file does not exist: throw `IllegalStateException` with message containing path and guidance ("Create a users.yaml file or set library.users-file")
   - If YAML is malformed: throw `IllegalStateException` with "invalid YAML" in message
   - If required fields missing: throw `IllegalStateException` listing which field is missing
   - All exceptions must be descriptive enough for ops troubleshooting

4. **Define the `UserEntry` record**
   - Nested public static record inside UsersFile, OR top-level in same package
   - Fields: `String username`, `String encodedPassword`, `List<String> roles`
   - Validate: username not blank, encodedPassword not blank, roles not empty
   - NOTE: Do NOT reuse the existing `com.agentlibrary.model.User` record (it lacks encodedPassword by design)

5. **Write unit tests**
   - File: `src/test/java/com/agentlibrary/auth/UsersFileTest.java`
   - Test: valid YAML loads correctly, returns expected entries
   - Test: missing file throws descriptive IllegalStateException
   - Test: malformed YAML throws descriptive exception
   - Test: missing username/password/roles field throws exception
   - Test: empty users list is valid (returns empty list)
   - Use `@TempDir` for file fixtures (same pattern as SecurityConfigYamlParseTest)

6. **Refactoring consideration for SecurityConfig (done in TASK-0006.02)**
   - The existing `loadUsersFromYaml` private method in SecurityConfig will be replaced by delegation to UsersFile in TASK-0006.02
   - Do NOT modify SecurityConfig in this task

### AC Mapping

- AC#2 (missing file throws descriptive exception) → Step 3, verified by test
- AC#3 (User record contains username, encodedPassword, roles) → Step 4

### Dependencies
