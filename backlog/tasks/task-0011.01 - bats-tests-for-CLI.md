## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
1. Create test-cli/ directory at project root.
2. Create test-cli/setup_suite.bash — bats helper that:
   a. Checks AGENT_LIB_SERVER env var (default http://localhost:8080)
   b. Sets AGENT_LIB_USER=admin, AGENT_LIB_PASS=changeme
   c. Exports SCRIPT_DIR pointing to ../scripts/
   d. Waits for server health (poll /actuator/health up to 30s)
3. Create test-cli/cli_basic.bats:
   a. @test "search returns seeded artifact" — run agent-lib search "agent-lib" and assert output contains "agent-lib-sh"
   b. @test "search with --type filter" — search with --type skill, verify results are all skills
   c. @test "list returns artifacts" — run agent-lib list, assert exit 0 and output has table headers + at least 1 row
   d. @test "list with --type filter" — run agent-lib list --type skill, verify only skill types
   e. @test "info shows metadata" — run agent-lib info agent-lib-sh, assert output contains name/type/version
   f. @test "pull downloads artifact file" — run agent-lib pull agent-lib-sh in a tmp dir, assert file created and non-empty
4. Create test-cli/cli_install.bats:
   a. @test "install places file at harness path" — run agent-lib install agent-lib-sh --harness claude --dest $BATS_TMPDIR/install-test (--fetch-only first), assert file exists at expected path
   b. @test "install --fetch-only extracts to ./artifact/" — run install with --fetch-only --dest $BATS_TMPDIR/fetch, assert directory has content
5. Create test-cli/cli_group.bats (AC #5, #6):
   a. @test "group install places all members at correct harness paths" — run agent-lib install dev-team-group --harness claude, assert each member file exists at expected target path
   b. @test "group dry-run lists all members without writing files" — run agent-lib install dev-team-group --dry-run, assert output lists all 3 members (manager, analyse, implementation) and NO files written to disk
6. Create test-cli/run-tests.sh — convenience wrapper:
   a. Starts docker compose (agent-library service) if not already running
   b. Waits for health
   c. Runs bats test-cli/*.bats
   d. Captures exit code
   e. Optionally tears down docker compose (--no-teardown flag)
   f. Exits with bats exit code
7. AC #4 (CI): Add a note in run-tests.sh header documenting CI usage — expects docker compose up already done by CI pipeline; just runs bats directly.
8. Ensure all bats tests use setup/teardown functions to create and clean tmp dirs (BATS_TMPDIR).
9. Tests must NOT depend on bats-assert or bats-support libraries (use plain [[ ]] and grep for portability). If bats-core is unavailable, run-tests.sh prints skip message and exits 0.
10. Files to create: test-cli/setup_suite.bash, test-cli/cli_basic.bats, test-cli/cli_install.bats, test-cli/cli_group.bats, test-cli/run-tests.sh
